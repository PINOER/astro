在日常开发、配置跨境电商 API、拼接营销追踪链接，或者在 CSS 中嵌入小图标时，我们经常会遇到两串特殊的“乱码”：

1. 一串由字母、数字和 `=` 结尾组成的文本：`SGVscE1pbmkgQUkgVG9vbHM=`
2. 一串带有大量 `%` 符号的网址：`https%3A%2F%2Fhelpmini.com%2F`

它们分别就是著名的 **Base64 编码** 和 **URL 编码（Percent-Encoding）**。

---

## 什么是 Base64 编码？

Base64 是一种用 **64 个可打印字符**（`A-Z`、`a-z`、`0-9`、`+`、`/`）来表示任意二进制数据的方式。

### 为什么需要 Base64？
传统的网络传输协议（如早期邮件协议或部分 HTTP 头信息）只支持传输 ASCII 可打印字符。如果你直接传输包含二进制数据的图片、文件，或者包含复杂编码的中文文本，很容易因为传输通道的字符过滤而乱码或损坏。

通过 Base64 转换：
- **图片内联（Data URI）**：将网页小图标（如 SVG、PNG）直接转为 Base64 写入 CSS，减少 HTTP 请求数。
- **凭证传输**：将 API Key、Basic Auth 用户名与密码拼装后转换传输。

> **⚠️ 注意**：Base64 是一种**编码方式（Encoding）**，而不是加密手段（Encryption）！任何人都可以毫无门槛地将其还原为原文。

---

## 什么是 URL 编码（Percent-Encoding）？

URL 网址只能包含特定的 **安全字符**（ASCII 字符集内的字母、数字及少量标点符号）。

当你在网址中传递中文、空格或特殊符号（如 `?`, `=`, `&`, `/`）时，为了防止浏览器和服务器产生解析歧义，必须将其转换为 `%` 加两位十六进制数字的形式。

### 常见字符编码对照：
- **空格** 转换为 `%20`
- **斜杠 `/`** 转换为 `%2F`
- **问号 `?`** 转换为 `%3F`
- **中文“帮助”** 转换为 `%E5%B8%AE%E5%8A%A9`
